# Riyobox Deployment Guide

## 1. Backend Deployment (Render)

### Option A: Docker Deployment (Recommended)
1. In Render, create a new **Web Service**.
2. Connect your GitHub repository.
3. Select the `riyobox-backend` directory.
4. Render will automatically detect the `Dockerfile`.
5. Add the following **Environment Variables**:
   - `MONGODB_URI`: Your MongoDB connection string.
   - `REDIS_HOST`: Your Redis instance address.
   - `REDIS_PORT`: Default is `6379`.
   - `REDIS_PASSWORD`: If applicable.
   - `JWT_SECRET`: A long random string for authentication.
   - `CORS_ALLOWED_ORIGINS`: Your web app URL (e.g., `https://riyobox-admin.pages.dev`).

### Option B: Native Java Deployment
1. Build Command: `mvn clean package -DskipTests` (inside `riyobox-backend`).
2. Start Command: `java -jar target/*.jar --spring.profiles.active=production`.

## 2. Admin Web Panel Deployment
1. Upload the `riyobox-admin` directory (e.g., to Vercel, Netlify, or GitHub Pages).
2. Set the Environment Variable:
   - `VITE_API_URL`: Your backend URL (e.g., `https://riyobox-backend.onrender.com/api`).
3. For local development, copy `.env.example` to `.env`.

## 3. Android APK Configuration
1. Open `riyobox-android/app/src/main/java/com/riyobox/data/network/RetrofitClient.kt`.
2. Update `PROD_BASE_URL` with your Render backend URL.
3. Push the changes. GitHub Actions will build a new APK.
4. Download the `riyobox-release-apk` from the GitHub Actions "Artifacts" section.

### Signing the APK (for Play Store)
The APK generated by GitHub Actions is currently **unsigned**. To make it a "real" app for the Play Store or to avoid security warnings:
1. Generate a Keystore in Android Studio.
2. Upload the Keystore to GitHub Secrets.
3. Update the GitHub Actions workflow to use `r0adkll/sign-android-release@v1`.
